# OpenCode Frontend Design

Use when building UI with OpenCode CLI.

## Rules

1. Use simple, focused components (OpenCode works best with clear boundaries)
2. Extract complex logic to hooks/utils
3. Use Tailwind CSS with simple, readable class names
4. Make small, atomic changes and review each one
5. Verify compilation after each change

## Anti-Patterns

- Complex refactoring in one shot
- Ambiguous prompts (be explicit)
- Ignoring compilation errors
- Bleeding-edge framework features
- Large, monolithic components
